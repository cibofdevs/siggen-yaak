# PAYOK Signature Generator — Yaak Plugin

A [Yaak HTTP Client](https://yaak.app) plugin that generates PAYOK digital signatures using **RSA-SHA256**, directly within your requests — no external tools needed.

---

## Features

| Template Function | Description |
|---|---|
| `payok.signature` | Generate an RSA-SHA256 signature from the request body and endpoint path |
| `payok.timestamp` | Generate the current ISO8601 timestamp |

### `payok.signature`

Signs the request using the formula:

```
message = <request_body> + "&" + <endpoint_path>
signature = RSA-SHA256-sign(privateKey, message) → Base64
```

> The plugin reads the request body automatically from the active request context — you do not need to copy the body into the signature arguments.

**Parameters (filled in via the Yaak dialog):**

| Name | Description |
|---|---|
| `path` | Endpoint path, e.g. `/api/v1/payment` |
| `private_key` | RSA private key in Base64 format without PEM headers, or an environment variable reference |

### `payok.timestamp`

Returns the current time as an ISO8601 string (e.g. `2024-01-15T10:30:00.123Z`).
The value is cached for 10 seconds so the timestamp in the body and in the signature always match within a single request cycle.

## Installation

### Prerequisites

- [Yaak](https://yaak.app) (latest version)
- [Node.js](https://nodejs.org) v18 or higher
- npm

### Steps

**1. Clone or download this plugin:**

```bash
git clone <repo-url> siggen-yaak
cd siggen-yaak
```

**2. Install dependencies:**

```bash
npm install
```

**3. Build the plugin:**

```bash
npm run build
```

**4. Add to Yaak:**

Open Yaak → **Settings** → **Plugins** → **Add Plugin** → point to the `siggen-yaak` folder.

## Usage

> **Important:** Do not type raw template syntax by hand. Use Yaak's autocomplete — start typing the function name and Yaak will suggest it, then a dialog will open for you to fill in the parameters.

### 1. Add the `sign` header

1. Open the **Headers** tab on your request
2. Set the header name to `sign`
3. Click the value field, start typing `payok` → Yaak shows autocomplete
4. Select **`payok.signature`** → the dialog opens
5. Fill in:
   - **Endpoint Path**: `/api-pay/payment/V3.2/order/create-api`
   - **Private Key**: type `PAYOK_PRIVATE_KEY` and select it from the env variable autocomplete
6. Click **Save**

The header value will appear as a `payok.signature(…)` chip.

Raw syntax generated by Yaak (for reference only):
```
${[ payok.signature(private_key=b64'JHtbIFBBWU9LX1BSSVZBVEVfS0VZIF19', path=b64'L2FwaS1wYXkvcGF5bWVudC9WMy4yL29yZGVyL2NyZWF0ZS1hcGk') ]}
```

### 2. Add `requestTime` to the body

In the JSON body editor, position your cursor inside the `requestTime` value, then:
1. Type `${[` or use Yaak's template tag button
2. Select **`payok.timestamp`** from the autocomplete
3. The tag will appear as an inline chip inside the body

Example body with a timestamp:
```
{"paymentMethodCode":"UPI","requestTime":"${[ payok.timestamp() ]}","amount":300,...}
```

### 3. Store the private key in an environment variable

1. Open **Environments** in Yaak
2. Add a variable:
   ```
   PAYOK_PRIVATE_KEY = MIIEvQIBADANBgkq...
   ```
3. When filling in the **Private Key** field in the `payok.signature` dialog, type the variable name and pick it from the autocomplete — Yaak encodes the reference automatically.

## Full Request Example

**Headers:**
```
Content-Type: application/json
sign: ${[ payok.signature(private_key=b64'JHtbIFBBWU9LX1BSSVZBVEVfS0VZIF19', path=b64'L2FwaS1wYXkvcGF5bWVudC9WMy4yL29yZGVyL2NyZWF0ZS1hcGk') ]}
```

**Body:**
```json
{"paymentMethodCode":"UPI","notificationUrl":"https://webhook-js.netlify.app/r/2345b60e-21f5-4627-9825-145a0c0fadfc","requestTime":"${[ payok.timestamp() ]}","amount":300,"merchantId":"420003","countryCode":"IN","currency":"INR","language":"EN","merchantOrderId":"6010095BNIVA1719477048730","goodsInfo":{"price":"300.00","name":"Jack TEST","id":"123456"},"customer":{"personalId":"1234567890","address":"Merchant Address","city":"Delhi","phone":"919876543210","ip":"89.32.16.47","name":"John","countryName":"India","deviceId":"DE-NO123456","email":"test@payok.com"},"customerAccount":{"orgCode":"IFSC","accountName":"Test","accountNumber":"1234567890"}}
```

> `b64'JHtbIFBBWU9LX1BSSVZBVEVfS0VZIF19'` is Yaak's encoding for `${[ PAYOK_PRIVATE_KEY ]}`.
> You never need to type this manually — Yaak generates it automatically when you select an env variable from the dialog.

## Private Key Formats

Both formats are accepted:

**Format 1 — Raw Base64 (no PEM headers):**
```
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC...
```

**Format 2 — Full PEM:**
```
-----BEGIN PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC...
-----END PRIVATE KEY-----
```

The plugin strips PEM headers automatically if present.

---

## Signing Algorithm

1. **Build the message:** `message = body + "&" + path`
2. **Wrap the private key** in PKCS#8 PEM headers if not already present
3. **Sign** using RSA with SHA-256 digest (PKCS#1 v1.5 padding)
4. **Encode** the resulting signature as Base64

Uses Node.js built-in `node:crypto` — no external dependencies required.

## Troubleshooting

**Plugin not showing in Yaak**
- Ensure the plugin folder contains `package.json` with `"main": "build/index.js"`
- Make sure you have run `npm run build`
- Restart Yaak

**Signature rejected by server**
- Check that the private key is PKCS#8, not PKCS#1 (PKCS#1 headers read `-----BEGIN RSA PRIVATE KEY-----`)
- Verify `path` is correct including the leading slash, e.g. `/api-pay/payment/V3.2/order/create-api`
- Make sure the request body type is JSON
- Use the Yaak dialog to fill in parameters — do not type raw syntax manually

**Converting PKCS#1 to PKCS#8**

```bash
openssl pkcs8 -topk8 -nocrypt -in rsa_private.pem -out pkcs8_private.pem
```
